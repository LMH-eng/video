## 项目经历
### 短视频平台（全栈）
- **项目描述**：基于 Qt + FFmpeg + SDL + epoll + MySQL + Redis + nginx-rtmp 实现短视频平台，覆盖注册登录、视频上传、播放、评论点赞、推荐等完整链路。
- **我的角色**：客户端与服务端联调、核心功能开发、性能与稳定性优化。

#### 核心工作与成果
- **架构与通信**：
  - 设计并实现客户端/服务端二进制协议通信链路，包含请求响应结构、长度头处理与路由分发。
  - 在服务端实现 `epoll + 线程池` 的并发处理框架，支持网络请求与业务处理解耦。
- **视频播放能力**：
  - 基于 FFmpeg + SDL 实现播放器三线程模型（读包/视频解码渲染/音频回调），完成音视频同步。
  - 支持本地文件与 RTMP 网络流播放，加入网络抖动下的缓冲与丢包策略。
- **倍速播放优化**：
  - 基于 `atempo` 实现音频倍速（0.5x~4.0x），并通过 `aformat + S16` 统一输出格式。
  - 排查并修复 1.5x/2.0x 下“电音叠加”问题（混音写入方式不当、滤镜链输出处理），显著提升听感稳定性。
- **上传与存储链路**：
  - 实现文件分块上传与 MD5 秒传校验，减少重复上传带宽开销。
  - 在客户端实现上传工作线程，避免 UI 阻塞，支持任务中断控制。
- **互动与推荐**：
  - 完成评论、点赞、列表展示等功能；服务端引入 Redis 做频控与兴趣数据缓存。
  - 参与推荐逻辑实现（兴趣向量 + 相似度思路），提升内容分发相关性。

#### 难点与解决（面试高频）
- **倍速电音问题定位**：
  - 现象：1.5x/2.0x 时出现“正常声音 + 电音”。
  - 排查：从日志与回调链路入手，逐步验证 SDL 输出、滤镜图、重采样路径。
  - 解决：改为直接拷贝音频数据、约束滤镜输出格式、清理不合理输出丢弃逻辑。
- **高并发上传稳定性**：
  - 现象：高频上传请求下出现处理顺序与吞吐抖动。
  - 解决：优化任务投递与处理策略，增强队列管理与线程协同。
