
# 录屏+画中画项目功能讲解与踩坑记录

### 一、项目概述

本项目是一个基于 Qt + FFmpeg + OpenCV 的**录屏+画中画**工具，核心功能：

- 采集桌面画面作为主视频
- 采集摄像头画面作为画中画（小窗）
- 将视频编码为 FLV 格式，支持保存到本地或推流到 RTMP 服务器

### 四、关键代码逻辑解析

#### 1. 编码器初始化流程（slot_setInfo）

```cpp
void SaveVideoFileThread::slot_setInfo(STRU_AV_FORMAT& format)
{
    this->m_avFormat = format;  // 保存参数（分辨率、帧率、码率、文件名）

    // 1. 创建输出上下文，指定 FLV 格式（兼容 RTMP 推流）
    avformat_alloc_output_context2(&oc, NULL, "flv", filename);

    // 2. 添加 H.264 视频流
    add_video_stream(&video_st, oc, &video_codec, AV_CODEC_ID_H264);

    // 3. 打开编码器，设置 superfast + zerolatency（实时编码）
    open_video(oc, video_codec, &video_st);

    // 4. 打开输出文件（本地文件或网络流）
    avio_open(&oc->pb, filename, AVIO_FLAG_WRITE);

    // 5. 写入文件头
    avformat_write_header(oc, NULL);
}
```

**为什么选 FLV 格式？** FLV 是 RTMP 协议的标准封装格式，同一个输出既可保存本地 `.flv` 文件，也可直接推流到 `rtmp://` 地址，零改动切换。

#### 2. 实时编码参数选择

```cpp
av_dict_set(&opt, "preset", "superfast", 0);
av_dict_set(&opt, "tune", "zerolatency", 0);
c->thread_count = 4;
```

- `preset superfast`：牺牲压缩率换取编码速度，实时录屏必须
- `tune zerolatency`：禁用前瞻等延迟优化，减少编码延迟
- `thread_count 4`：多线程编码，充分利用 CPU

#### 3. RGB → YUV 转换（ImageToYuvBuffer）

FFmpeg 编码器要求输入 YUV420P 格式，而 Qt 截图得到的是 RGB24，必须转换：

```cpp
SwsContext *rgb_to_yuv_ctx = sws_getContext(
    w, h, AV_PIX_FMT_RGB24,    // 源：RGB24
    w, h, AV_PIX_FMT_YUV420P,  // 目标：YUV420P
    SWS_BICUBIC, NULL, NULL, NULL);
sws_scale(rgb_to_yuv_ctx, ...);
```

#### 4. 画中画窗口实现

```cpp
// PictureWidget 构造函数
this->setWindowFlags(Qt::FramelessWindowHint | Qt::WindowStaysOnTopHint);
```

- `FramelessWindowHint`：无边框，不显示标题栏
- `WindowStaysOnTopHint`：始终置顶，悬浮在所有窗口之上

### 五、踩坑记录：项目运行崩溃问题排查与修复

#### 问题现象

程序运行后点击"停止"按钮直接崩溃，且视频文件没有写入任何数据。

---

#### Bug 1（崩溃根因）：slot_setInfo 参数名遮蔽成员变量

**位置**：`savevideofilethread.cpp` 的 `slot_setInfo`

**原始代码**：
```cpp
void SaveVideoFileThread::slot_setInfo(STRU_AV_FORMAT& m_avFormat)  // ← 参数名和成员变量同名！
{
    m_avFormat = avFormat();  // ← 方向反了！
```

**问题分析**：

参数名 `m_avFormat` 与类成员变量 `m_avFormat` 同名，函数体内的 `m_avFormat` 指的是参数而非成员。`avFormat()` 返回的是成员 `this->m_avFormat`，所以这行代码的实际效果是：

```
参数（新值） = 成员（旧值）    // 方向完全反了！
```

成员变量永远不会被更新，`fileName` 始终为空字符串 → `avio_open` 打开空路径失败 → `slot_setInfo` 提前 return → `oc` 已分配但未完成初始化 → 点击"停止"时 `av_write_trailer(oc)` 操作未初始化的上下文 → **崩溃**。

**修复**：
```cpp
void SaveVideoFileThread::slot_setInfo(STRU_AV_FORMAT& format)
{
    this->m_avFormat = format;  // 成员 = 参数，方向正确
```

**教训**：参数名不要和成员变量同名，避免遮蔽（shadowing）。如果必须同名，用 `this->` 显式区分。

---

#### Bug 2：slot_closeVideo 缺少空指针保护

**位置**：`savevideofilethread.cpp` 的 `slot_closeVideo`

**原始代码**：
```cpp
void SaveVideoFileThread::slot_closeVideo()
{
    m_picInPicRead->slot_closeVideo();
    av_write_trailer(oc);  // ← 如果 oc 为空直接崩溃
```

**问题分析**：

如果用户没有点"开始"就点"停止"，或者 `slot_setInfo` 因文件路径为空而提前 return，`oc` 可能是未初始化的指针或 nullptr，直接调用 `av_write_trailer(oc)` 会崩溃。

**修复**：
```cpp
void SaveVideoFileThread::slot_closeVideo()
{
    m_picInPicRead->slot_closeVideo();
    if (!oc) return;  // 空指针保护
    av_write_trailer(oc);
    // ... 释放资源 ...
    oc = nullptr;  // 防止重复释放
}
```

#### Bug 3：视频帧写入代码被注释，导致视频文件无数据

**位置**：`picinpic_read.cpp` 的 `slot_getVideoFrame`

**原始代码**：
```cpp
//    uint8_t * picture_buf = NULL;
//    int buffer_size = ImageToYuvBuffer( image , &picture_buf );
//    Q_EMIT SIG_sendVideoFrameData( picture_buf, buffer_size );  // ← 被注释掉了！
    Q_EMIT SIG_sendVideoFrame( image );  // ← 只发了预览信号
```

**问题分析**：

`SIG_sendVideoFrameData` 信号被注释掉后，`SaveVideoFileThread::slot_writeVideoFrameData` 永远不会被调用，编码器收不到任何帧数据，视频文件为空。`SIG_sendVideoFrame` 只用于界面预览，不触发写入。

**修复**：取消注释，恢复视频帧写入：
```cpp
uint8_t * picture_buf = NULL;
int buffer_size = ImageToYuvBuffer( image , &picture_buf );
Q_EMIT SIG_sendVideoFrameData( picture_buf, buffer_size );
Q_EMIT SIG_sendVideoFrame( image );
```

#### Bug 4：setWindowFlags 覆盖了窗口类型标志

**位置**：`recorderdialog.cpp` 构造函数

**原始代码**：
```cpp
this->setWindowFlags(Qt::WindowMinMaxButtonsHint | Qt::WindowCloseButtonHint);
```

**问题分析**：

`setWindowFlags()` 是**替换式**设置，不是追加式。这行代码把 `RecorderDialog` 默认的 `Qt::Dialog` 窗口类型完全覆盖了，只剩两个装饰性 Hint。没有窗口类型标志，Qt 把它当作普通子控件，`w.show()` 不会弹出独立窗口。

**修复**：
```cpp
this->setWindowFlags(this->windowFlags() | Qt::WindowMinMaxButtonsHint);
```

用 `windowFlags() |` 追加标志，保留原有的 `Qt::Dialog` 类型。

---

#### Bug 5：成员指针未初始化

**位置**：`savevideofilethread.h`

**原始代码**：
```cpp
AVFormatContext *oc;        // 未初始化
AVCodec *video_codec;       // 未初始化
AVOutputFormat *fmt;        // 未初始化
```

**问题分析**：

未初始化的指针值是随机的，`if (!oc)` 判断不可靠，可能误判为非空导致访问非法内存。

**修复**：
```cpp
AVFormatContext *oc = nullptr;
AVCodec *video_codec = nullptr;
AVOutputFormat *fmt = nullptr;
```

#### Bug 6：路径替换破坏 RTMP 推流地址

**位置**：`recorderdialog.cpp` 的 `on_pb_setUrl_clicked`

**原始代码**：
```cpp
m_saveUrl = ui->le_url->text();
m_saveUrl = m_saveUrl.replace("/", "\\");  // 无条件替换
```

**问题分析**：

如果用户输入的是 RTMP 推流地址（如 `rtmp://192.168.18.129/videotest`），替换后变成 `rtmp:\\192.168.18.129\videotest`，地址格式被破坏，推流失败。

**修复**：
```cpp
m_saveUrl = ui->le_url->text();
if (!m_saveUrl.startsWith("rtmp://", Qt::CaseInsensitive)) {
    m_saveUrl = m_saveUrl.replace("/", "\\");
}
```

只对本地文件路径做反斜杠替换，RTMP 地址保持原样。

### 六、DLL 部署注意事项

程序依赖 FFmpeg 和 OpenCV 的 DLL，运行时必须确保 DLL 与 exe 在**同一目录**下。

**常见错误**：把 DLL 放在 exe 的子目录（如 `debug/dll/`），Windows 不会搜索子目录，导致程序启动时找不到 DLL 直接崩溃，且不弹出任何错误提示。

**正确做法**：
```
build-debug/debug/
  ├── VideoRecorder.exe
  ├── avcodec-58.dll        ← DLL 必须和 exe 同级
  ├── avformat-58.dll
  ├── libopencv_core420.dll
  └── ... 其他所有 DLL
```

### 七、面试高频问题

#### Q1：录屏项目中为什么用 YUV420P 而不是 RGB24？

**答**：H.264 编码器标准要求输入 YUV420P 格式。YUV420P 相比 RGB24 有两个优势：
1. **数据量减半**：YUV420P 对色度分量 4:2:0 子采样，每 4 个亮度像素共享一组色度，数据量是 RGB24 的 50%
2. **人眼感知匹配**：人眼对亮度敏感、对色度不敏感，子采样后视觉损失极小

#### Q2：录屏和直播推流的区别？代码层面怎么切换？

**答**：录屏保存到本地文件，直播推流发送到 RTMP 服务器。代码层面**零改动**，只需修改 `fileName`：
- 录屏：`fileName = "D:\\record.flv"`
- 推流：`fileName = "rtmp://192.168.18.129/videotest"`

FFmpeg 的 `avio_open` 自动识别协议，本地路径走文件 I/O，`rtmp://` 走网络推流。

#### Q3：画中画窗口如何实现始终置顶？

**答**：通过 Qt 的窗口标志 `Qt::WindowStaysOnTopHint`，让窗口在 Z 轴上始终位于最顶层。配合 `Qt::FramelessWindowHint` 去掉标题栏，实现轻量悬浮窗效果。

#### Q4：setWindowFlags 的替换式设置和追加式设置有什么区别？

**答**：
- `setWindowFlags(A | B)`：**替换**，把窗口标志设为 `A | B`，原来的标志全部丢失
- `setWindowFlags(windowFlags() | A)`：**追加**，在原有标志基础上添加 `A`

常见错误是用替换式设置只传 Hint 标志（如 `WindowMinMaxButtonsHint`），丢失了窗口类型标志（如 `Dialog`、`Window`），导致窗口无法正常显示。

#### Q5：参数名遮蔽（shadowing）是什么？如何避免？

**答**：当函数参数名与成员变量同名时，函数体内的名字指向参数而非成员，成员变量被"遮蔽"。例如 `slot_setInfo(STRU_AV_FORMAT& m_avFormat)` 中 `m_avFormat` 指参数不指成员。避免方法：
1. 参数用不同名字（如 `format`）
2. 访问成员时用 `this->m_avFormat` 显式指定
3. 成员变量统一加前缀 `m_`，参数不加前缀
