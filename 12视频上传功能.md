# 12 视频上传功能实现

## 一、功能概述

实现短视频平台的视频上传功能，客户端选择本地 mp4/flv 视频，自动生成 GIF 缩略图，通过多线程 TCP 分片上传到服务端，服务端接收完成后写入数据库，支持 MD5 去重、视频分类标签、上传进度显示、重复上传拦截。

## 二、整体架构

```
客户端(UploadDialog → MainDialog → CKernel → UploadWorker线程 → netapi TCP)
    ↓ STRU_UPLOAD_RQ（上传请求包头）
    ↓ STRU_FILEBLOCK_RQ × N（循环分片发送）
服务端(Block_Epoll_Net → TcpKernel → CLogic → MySQL)
    ↓ 文件接收 + 重命名 + DB INSERT
RTMP流媒体点播（nginx-rtmp）
```

### 上传请求

```cpp
#define _DEF_PACK_UPLOAD_RQ     (_DEF_PACK_BASE + 4)
#define _DEF_PACK_UPLOAD_RS     (_DEF_PACK_BASE + 5)
#define _DEF_PACK_FILEBLOCK_RQ  (_DEF_PACK_BASE + 6)
#define _DEF_PACK_FILEBLOCK_RS  (_DEF_PACK_BASE + 7)
#define _DEF_PACK_RECOMMEND_RQ  (_DEF_PACK_BASE + 8)
#define _DEF_PACK_RECOMMEND_RS  (_DEF_PACK_BASE + 9)

// 上传请求包头
typedef struct STRU_UPLOAD_RQ
{
    STRU_UPLOAD_RQ()
    {
        m_nType = _DEF_PACK_UPLOAD_RQ;
        m_nFileId = 0;
        m_UserId = 0;
        m_nFileSize = 0;
        memset(m_szFileName, 0, sizeof(m_szFileName));
        memset(m_szGifName, 0, sizeof(m_szGifName));
        memset(m_szFileType, 0, sizeof(m_szFileType));
        memset(m_szHobby, 0, sizeof(m_szHobby));
        memset(m_szFileHash, 0, sizeof(m_szFileHash));
    }
    PackType m_nType;              // 协议类型
    int m_UserId;                  // 登录用户ID
    int m_nFileId;                 // 文件随机唯一ID（区分上传任务）
    int64_t m_nFileSize;           // 文件总字节数
    char m_szHobby[8];             // 8字节Hobby分类标签
    char m_szFileName[260];        // 视频文件名
    char m_szGifName[260];         // GIF缩略图文件名
    char m_szFileType[40];         // 文件类型后缀（mp4/flv/gif）
    char m_szFileHash[33];         // MD5哈希（32字符+'\0'）
}STRU_UPLOAD_RQ;

// 上传回复
typedef struct STRU_UPLOAD_RS
{
    STRU_UPLOAD_RS() : type(_DEF_PACK_UPLOAD_RS), result(upload_refuse) {}
    PackType type;
    int result;  // 0=拒绝 1=同意 2=视频已存在（MD5重复）
}STRU_UPLOAD_RS;

// 文件块请求
typedef struct STRU_FILEBLOCK_RQ
{
    STRU_FILEBLOCK_RQ()
    {
        type = _DEF_PACK_FILEBLOCK_RQ;
        m_nFileId = 0;
        m_nBlockIndex = 0;
        m_nBlockSize = 0;
        memset(m_szBlockData, 0, sizeof(m_szBlockData));
    }
    PackType type;
    int m_nFileId;                 // 对应上传请求的文件ID
    int m_nBlockIndex;             // 文件块序号
    int m_nBlockSize;              // 实际数据大小
    char m_szBlockData[4096];      // 文件块数据
}STRU_FILEBLOCK_RQ;

// 文件块回复
typedef struct STRU_FILEBLOCK_RS
{
    STRU_FILEBLOCK_RS() : type(_DEF_PACK_FILEBLOCK_RS), m_nFileId(0), m_nBlockIndex(0), result(fileblock_fail) {}
    PackType type;
    int m_nFileId;
    int m_nBlockIndex;
    int result;  // 0=失败 1=成功
}STRU_FILEBLOCK_RS;
```

## 五、客户端实现

### 5.1 UploadWorker（上传工作线程类）

**文件**：`uploadworker.h` / `uploadworker.cpp`

**设计思路**：使用 `moveToThread` 将上传工作对象移入子线程，避免大文件上传阻塞UI主线程。

**核心方法**：

```cpp
// 上传入口：先上传视频，再上传GIF
void UploadWorker::slot_doUpload(QString filePath, QString imgPath, Hobby hy, int userId)
{
    m_bCancelled = 0;  // 重置取消标志

    // 计算总上传大小（视频+GIF）
    qint64 videoSize = QFileInfo(filePath).size();
    qint64 gifSize = QFileInfo(imgPath).size();
    qint64 totalAllSize = videoSize + gifSize;
    qint64 allUploaded = 0;

    // 1. 先上传视频文件（携带GIF文件名信息）
    qint64 videoUploaded = UploadFile(filePath, hy, userId,
                                       QFileInfo(imgPath).fileName(),
                                       allUploaded, totalAllSize);

    // 如果被取消（服务端拒绝重复视频）则不再上传GIF
    if(m_bCancelled) {
        Q_EMIT SIG_uploadFinish(0);
        return;
    }

    allUploaded += videoUploaded;

    // 2. 等待服务端处理完成（避免上传状态竞态）
    QThread::msleep(500);

    // 3. 再上传GIF缩略图
    allUploaded += UploadFile(imgPath, hy, userId, "",
                               allUploaded, totalAllSize);

    Q_EMIT SIG_uploadFinish(1);
}
```

```cpp
// 上传单个文件（视频或GIF）
qint64 UploadWorker::UploadFile(QString filePath, Hobby hy, int userId,
                                 QString gifName, qint64 alreadyUploaded, qint64 totalAllSize)
{
    // 1. 封装STRU_UPLOAD_RQ上传包头
    STRU_UPLOAD_RQ rq;
    rq.m_nFileId = qrand() % 10000;  // 随机文件ID
    rq.m_UserId = userId;
    rq.m_nFileSize = info.size();
    strcpy_s(rq.m_szFileName, _MAX_FILE_PATH, fileName.toLocal8Bit().data());

    // 截取文件后缀
    QString suffix = filePath.right(filePath.length() - filePath.lastIndexOf('.') - 1);
    memcpy(rq.m_szFileType, suffix.toLocal8Bit().data(), suffix.length());

    // 拷贝GIF文件名（仅视频上传时携带）
    if(!gifName.isEmpty()) {
        strcpy_s(rq.m_szGifName, _MAX_FILE_PATH, gifName.toLocal8Bit().data());
    }

    // 拷贝Hobby标签
    memcpy(rq.m_szHobby, &hy, sizeof(Hobby));

    // 计算文件MD5哈希（用于服务端去重）
    QCryptographicHash md5Hash(QCryptographicHash::Md5);
    QFile hashFile(filePath);
    if(hashFile.open(QIODevice::ReadOnly)) {
        while(!hashFile.atEnd()) {
            md5Hash.addData(hashFile.read(8192));
        }
        hashFile.close();
        QByteArray hashResult = md5Hash.result().toHex();
        strcpy_s(rq.m_szFileHash, 33, hashResult.data());
    }

    // TCP发送上传请求包头
    m_tcpClient->SendData(0, (char*)&rq, sizeof(rq));

    // 2. 循环分片读取文件并发送
    QFile file(filePath);
    file.open(QIODevice::ReadOnly);
    while(!file.atEnd()) {
        if(m_bCancelled) { file.close(); return 0; }  // 支持取消

        STRU_FILEBLOCK_RQ blockRq;
        blockRq.m_nFileId = rq.m_nFileId;
        blockRq.m_nBlockIndex = blockIndex;
        int readLen = file.read(blockRq.m_szBlockData, _DEF_BUFFER);
        if(readLen <= 0) break;
        blockRq.m_nBlockSize = readLen;

        m_tcpClient->SendData(0, (char*)&blockRq, sizeof(blockRq));
        uploadedSize += readLen;
        blockIndex++;
        Q_EMIT SIG_progress(alreadyUploaded + uploadedSize, totalAllSize);
        QThread::msleep(10);  // 限速，避免服务端处理不过来
    }
    file.close();
    return uploadedSize;
}
```

### 6.2 CLogic::UploadRq（上传请求处理）

```cpp
void CLogic::UploadRq(sock_fd clientfd, char* szbuf, int nlen)
{
    STRU_UPLOAD_RQ* rq = (STRU_UPLOAD_RQ*)szbuf;
    STRU_UPLOAD_RS rs;

    // 判断是否为GIF缩略图（GIF不做重复检查）
    bool isGifUpload = (strcmp(rq->m_szFileType, "gif") == 0);

    // 视频文件：通过MD5哈希检查是否已存在（去重）
    if(!isGifUpload && strlen(rq->m_szFileHash) > 0) {
        char sqlBuf[4096] = "";
        sprintf(sqlBuf, "SELECT id FROM t_VideoData WHERE hash='%s';", rq->m_szFileHash);
        list<string> lstRes;
        if(m_sql->SelectMysql(sqlBuf, 1, lstRes) && lstRes.size() > 0) {
            rs.result = video_already_exist;
            SendData(clientfd, (char*)&rs, sizeof(rs));
            return;
        }
    }

    // 创建视频存储目录
    system("mkdir -p /home/lmh/netdisk/videos");

    // 保存上传状态
    UploadInfo info;
    info.fileId = rq->m_nFileId;
    info.clientfd = clientfd;       // 保存客户端fd
    info.userId = rq->m_UserId;
    info.fileSize = rq->m_nFileSize;
    strcpy(info.fileName, rq->m_szFileName);
    strcpy(info.gifName, rq->m_szGifName);
    strcpy(info.fileType, rq->m_szFileType);
    strcpy(info.fileHash, rq->m_szFileHash);
    memcpy(info.hobby, rq->m_szHobby, 8);

    // 创建临时文件
    char tmpPath[520] = "";
    sprintf(tmpPath, "/home/lmh/netdisk/videos/%d_%s.tmp", rq->m_nFileId, rq->m_szFileName);
    info.fd = open(tmpPath, O_CREAT | O_WRONLY | O_TRUNC, 0644);
    if(info.fd < 0) {
        rs.result = upload_refuse;
        SendData(clientfd, (char*)&rs, sizeof(rs));
        return;
    }

    // 记录上传状态（key=fileId，视频和GIF各自独立）
    {
        lock_guard<mutex> lock(m_mapMutex);
        m_mapUpload[rq->m_nFileId] = info;
    }

    rs.result = upload_accept;
    SendData(clientfd, (char*)&rs, sizeof(rs));
}
```

### 6.3 CLogic::FileBlockRq（文件块上传处理）

```cpp
void CLogic::FileBlockRq(sock_fd clientfd, char* szbuf, int nlen)
{
    STRU_FILEBLOCK_RQ* rq = (STRU_FILEBLOCK_RQ*)szbuf;
    STRU_FILEBLOCK_RS rs;
    rs.m_nFileId = rq->m_nFileId;
    rs.m_nBlockIndex = rq->m_nBlockIndex;

    // 通过fileId查找上传状态
    UploadInfo info;
    {
        lock_guard<mutex> lock(m_mapMutex);
        auto it = m_mapUpload.find(rq->m_nFileId);
        if(it == m_mapUpload.end()) {
            rs.result = fileblock_fail;
            SendData(clientfd, (char*)&rs, sizeof(rs));
            return;
        }
        info = it->second;  // 拷贝出来，锁外做IO
    }

    // 写入文件块（锁外做IO）
    int writeLen = write(info.fd, rq->m_szBlockData, rq->m_nBlockSize);
    if(writeLen != rq->m_nBlockSize) {
        rs.result = fileblock_fail;
        SendData(info.clientfd, (char*)&rs, sizeof(rs));
        return;
    }

    info.uploadedSize += writeLen;
    info.blockIndex++;
    rs.result = fileblock_success;
    SendData(info.clientfd, (char*)&rs, sizeof(rs));

    // 上传完成
    if(info.uploadedSize >= info.fileSize) {
        close(info.fd);

        bool isGifUpload = (strcmp(info.fileType, "gif") == 0);
        if(isGifUpload) {
            // GIF：重命名保存，不写数据库
            rename(tmpPath, dstPath);
        } else {
            // 视频：重命名保存 + 拼接RTMP地址 + 写入数据库
            rename(tmpPath, dstPath);
            char rtmpUrl[520] = "";
            sprintf(rtmpUrl, "rtmp://192.168.18.129:1935/vod/%d_%s", info.fileId, info.fileName);
            char gifPath[520] = "";
            sprintf(gifPath, "/home/lmh/netdisk/videos/%s", info.gifName);

            // INSERT INTO t_VideoData(...)
            char sqlBuf[8192] = "";
            sprintf(sqlBuf, "INSERT INTO t_VideoData(userid, name, url, gif, hash, "
                    "food, funny, ennergy, dance, music, video, outside, edu) "
                    "VALUES(%d, '%s', '%s', '%s', '%s', %d, %d, %d, %d, %d, %d, %d, %d);",
                    info.userId, info.fileName, rtmpUrl, gifPath, info.fileHash,
                    info.hobby[0], info.hobby[1], info.hobby[2], info.hobby[3],
                    info.hobby[4], info.hobby[5], info.hobby[6], info.hobby[7]);
            m_sql->UpdataMysql(sqlBuf);
        }

        // 移除上传状态
        lock_guard<mutex> lock(m_mapMutex);
        m_mapUpload.erase(info.fileId);
    } else {
        // 未完成，更新进度
        lock_guard<mutex> lock(m_mapMutex);
        auto it = m_mapUpload.find(rq->m_nFileId);
        if(it != m_mapUpload.end()) {
            it->second.uploadedSize = info.uploadedSize;
            it->second.blockIndex = info.blockIndex;
        }
    }
}
```

## 七、关键Bug修复记录

### 7.1 上传状态map的key覆盖问题（原始Bug）

**现象**：视频上传必须上传两次才能在推荐列表中显示。

**根因**：服务端 `m_mapUpload` 使用 `clientfd` 作为key，同一客户端的视频上传和GIF上传共用同一个key。当GIF的 `UploadRq` 到达时，**覆盖**了视频上传的map条目，导致视频的 DB INSERT 被跳过。

**修复**：
- `m_mapUpload` 的 key 从 `clientfd` 改为 `fileId`
- `UploadInfo` 新增 `int clientfd` 字段保存客户端fd
- `FileBlockRq` 改用 `rq->m_nFileId` 查找map

### 7.2 服务端线程池乱序处理Bug（核心Bug）

**现象**：修改key后仍然上传失败，视频始终不出现在推荐列表。

**根因**：服务端 `recv_task` 接收TCP数据后，将处理任务投递到线程池异步执行。多个数据包（UploadRq、FileBlockRq）被不同工作线程并行处理，**TCP接收有序但处理无序**。

```
recv_task 线程A：读完 UploadRq → 投递 Buffer_Deal_A
recv_task 线程A：读完 FileBlockRq → 投递 Buffer_Deal_B

如果 Buffer_Deal_B 先执行 → FileBlockRq 找不到 fileId → 返回 fail
→ 视频文件只写了一部分，永远不会触发 DB INSERT
```

**修复**（`block_epoll_net.cpp`）：
```cpp
// 修改前：异步投递到线程池，处理顺序不可控
pthis->m_threadpool->Producer_add(Buffer_Deal, (void*)buffer);

// 修改后：同步调用，保证同一fd的包按接收顺序串行处理
Buffer_Deal((void*)buffer);
```

### 7.3 并发安全：mutex保护

**问题**：不同客户端的请求仍然并发执行，map的读写需要互斥保护。

**修复**：
- `clogic.h` 新增 `std::mutex m_mapMutex`
- `UploadRq`：写map时加 `lock_guard<mutex>`
- `FileBlockRq`：读map时加锁拷贝，IO操作在锁外执行，完成后再次加锁更新/删除

## 八、TCP数据包格式

```
发送端（SendData）：
+--------------+------------------+
|<-- 4bytes -->|<-- nlen -------->|
|   packsize   |   协议结构体     |
+--------------+------------------+

接收端（recv_task）：
1. read(fd, &nPackSize, 4)          // 先读4字节包长度
2. recv(fd, pSzBuf, nPackSize)      // 循环读满包体
3. Buffer_Deal() → DealData()       // 解析PackType，调用处理函数
4. ev->eventadd()                    // 重新注册EPOLLIN|EPOLLONESHOT
```

## 十二、踩坑记录

1. **m_mapUpload的key覆盖**：使用clientfd作为key，同一客户端视频和GIF上传共用key导致覆盖。修复：改用fileId作为key，各自独立条目。
2. **线程池乱序处理**：recv_task将Buffer_Deal异步投递到线程池，UploadRq和FileBlockRq可能被并行处理导致乱序。修复：改为同步调用Buffer_Deal，保证同一fd的包按接收顺序处理。
3. **FileBlockRq持锁做IO**：最初对整个函数持锁，send()阻塞时其他客户端被饿死。修复：只在读写map时持锁，IO操作在锁外执行。
4. **登录后不自动刷新**：MainDialog显示后推荐视频为空，需要手动点击。修复：登录成功后自动调用slot_recommendRq(m_userid)。
5. **上传完成信号重复触发**：ckernel直接调用slot_uploadFinish和上传线程信号都会触发。修复：使用m_uploadFinished标志防止重复调用。
6. **GIF上传需要等待**：视频上传完后立即上传GIF可能因服务端还在处理而竞态。修复：视频上传完成后QThread::msleep(500)等待服务端处理完毕。
