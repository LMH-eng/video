# 13 视频播放优化与推荐视频展示

## 一、功能概述

在视频上传功能（第12篇）完成后，继续完善短视频平台的客户端体验：
1. **推荐视频GIF网格展示**：从服务端获取推荐视频列表，HTTP异步下载GIF缩略图，在网格中展示，鼠标悬停播放GIF动画，点击直接播放RTMP视频
2. **视频播放器进度条修复**：修复RTMP流播放时向前拖动进度条卡死、切换视频卡死、频繁seek卡死等多个严重Bug

4. **UI比例修正**：推荐视频网格列宽等分、GIF缩略图保持原始比例居中（不拉伸变形）

### 2.1 自定义控件 QMyMovieLabel

**文件**：`mymovielabel.h` / `mymovielabel.cpp` / `mymovielabel.ui`

每个网格格子是一个 `QMyMovieLabel` 控件，内部包含一个 `QLabel`，支持：
- 加载GIF动画，默认暂停
- 鼠标移入（`enterEvent`）自动播放GIF
- 鼠标移出（`leaveEvent`）停止GIF
- 点击发射 `SIG_labelClicked()` 信号
- 缓存RTMP播放地址 `m_rtmpUrl`

```cpp
// GIF帧变化时按比例缩放显示（不拉伸变形）
void QMyMovieLabel::onMovieFrameChanged()
{
    if(!m_movie) return;
    QPixmap pix = m_movie->currentPixmap();
    if(pix.isNull()) return;
    // 按比例缩放居中显示
    QPixmap scaled = pix.scaled(ui->lb_movie->size(),
        Qt::KeepAspectRatio, Qt::SmoothTransformation);
    ui->lb_movie->setPixmap(scaled);
}
```

**关键设计**：不使用 `QLabel::setScaledContents(true)`（会拉伸变形），而是通过 `QMovie::frameChanged` 信号，每帧手动用 `Qt::KeepAspectRatio` 缩放后 `setPixmap`，保持视频原始比例，居中显示在格子中。

## 三、视频播放器进度条修复

### 3.1 Bug总览

| Bug | 现象 | 根因 |
|-----|------|------|
| 向前拖动卡死 | 进度条向前拖后画面冻结 | seek时未重置audio_clock，视频线程死等音频时钟追赶 |
| seek失败冻结 | seek失败后视频完全不动 | seek_flag在if/else外设置，失败也生效 |
| 同步死等 | 偶发性无限等待 | 音频同步循环无超时 |
| 切换视频卡死 | 点击另一个GIF程序冻结 | 无interrupt_callback，av_read_frame阻塞不被quit中断 |
| 频繁seek卡死 | 快速连续拖动进度条 | 无防抖，每次释放都触发RTMP seek |
| 滑块回弹 | 点击轨道后手柄先回旧位置再跳新位置 | 500ms定时器用旧播放位置拉回手柄 |

### 3.2 FFmpeg中断回调（核心修复）

**文件**：`videoplayer.h` / `videoplayer.cpp`

**问题**：播放RTMP网络流时，`av_read_frame`、`av_seek_frame`、`avformat_open_input` 都是网络I/O操作，可能阻塞几十秒。`stop()` 设置 `quit=true` 后，FFmpeg 根本不检查这个标志，继续阻塞，导致主线程 `wait()` 永久冻结。

**修复**：注册 FFmpeg 的 `interrupt_callback`，让所有阻塞 I/O 操作可以被 `quit` 标志中断。

```cpp
// 全局中断回调函数
int decode_interrupt_cb(void *ctx)
{
    VideoState *is = (VideoState *)ctx;
    return (is && is->quit) ? 1 : 0;  // quit时返回1，FFmpeg立即中断
}

// run()中在avformat_open_input之前设置
pFormatCtx->interrupt_callback.callback = decode_interrupt_cb;
pFormatCtx->interrupt_callback.opaque = &m_videoState;
```

设置后，`stop()` → `quit=true` → FFmpeg 立即中断所有阻塞 I/O → `run()` 几十毫秒内退出 → `wait()` 不再卡死。

### 3.3 seek时重置audio_clock

**文件**：`videoplayer.cpp`

**问题**：seek时只重置了 `video_clock = 0`，`audio_clock` 保持旧值。向前seek（如5s→60s）后，视频线程的同步等待循环 `video_pts(60) <= audio_pts(5)` 永远不成立 → 死等卡死。

**修复**：
```cpp
// seek成功后重置audio_clock到目标位置
m_videoState.audio_clock = (double)m_videoState.seek_pos;
m_videoState.seek_time = m_videoState.seek_pos;
m_videoState.seek_flag_audio = 1;
m_videoState.seek_flag_video = 1;
```

**为什么向后拖不卡死而向前拖卡死**：
- 向后seek（60s→5s）：`video_pts(5) <= audio_pts(60)` → 立即成立 → 不等待
- 向前seek（5s→60s）：`video_pts(60) <= audio_pts(5)` → 永远不成立 → 死等

### 3.4 seek失败不设置seek_flag

**问题**：原代码中 `seek_flag_audio = 1` 和 `seek_flag_video = 1` 在 `if/else` 外面，无论 `av_seek_frame` 成功失败都执行。RTMP流seek失败时，所有帧的PTS都 < seek_time，全部被跳过 → 视频冻结。

**修复**：将 seek_flag 设置移到 `else` 分支（成功时），失败时清零flag从当前位置继续播放。

### 3.5 同步等待循环超时保护

**文件**：`videoplayer.cpp`

```cpp
// 等待音频时钟（带超时保护）
if (is->audioq != NULL) {
    int syncWaitCount = 0;
    while(1) {
        if(is->quit) break;
        if(is->audioq->size == 0) break;
        double audio_pts = is->audio_clock;
        double v_pts = is->video_clock;
        if(v_pts <= audio_pts) break;
        // 超时保护：最多等待2秒（400 × 5ms = 2000ms）
        if(++syncWaitCount > 400) break;
        SDL_Delay(5);
    }
}
```

### 3.6 seek防抖（PlayerDialog层）

**文件**：`playerdialog.h` / `playerdialog.cpp`

**问题**：快速连续拖动进度条时，每次释放都调 `seek()` → 读线程调 `av_seek_frame()`（RTMP网络操作）→ 频繁seek导致服务器压力和视频卡顿。

**修复**：添加300ms防抖定时器，快速拖动时只有最后一次释放后300ms才真正执行seek。

```cpp
// playerdialog.h
QTimer *m_seekTimer;     // seek防抖定时器
qint64 m_pendingSeekPos; // 待执行的seek位置(微秒)

// playerdialog.cpp - 释放进度条
void PlayerDialog::slot_sliderReleased()
{
    m_sliderPressed = false;
    if(!isStop) {
        m_pendingSeekPos = (qint64)ui->slider_videoProcess->value() * 1000000;
        m_seekTimer->start();  // 300ms后执行
    }
}

// 定时器触发时才真正seek
void PlayerDialog::slot_seekTimeout()
{
    if(!isStop)
        m_player->seek(m_pendingSeekPos);
}
```

### 3.7 滑块手柄回弹修复

**文件**：`playerdialog.h` / `playerdialog.cpp`

**问题**：`slot_TimerTimeOut` 每500ms用实际播放位置 `getCurrentTime()` 调 `setValue()`。点击轨道后到seek真正生效之间，定时器会把滑块手柄拉回旧播放位置，然后seek完成后才跳到新位置 → 视觉上"先回原位再跳到点击位置"。

**修复**：添加 `m_seeking` 标志，seek期间定时器把手柄保持在目标位置而非拉回旧位置。

```cpp
// playerdialog.h
bool m_seeking;          // seek进行中标志
int m_seekTicks;         // seek已等待的定时器周期数（超时保护）

// playerdialog.cpp - 定时器逻辑
void PlayerDialog::slot_TimerTimeOut()
{
    if(m_sliderPressed) return;

    // seek进行中：把手柄保持在目标位置，不被实际播放位置拉回
    if(m_seeking) {
        qint64 targetSec = m_pendingSeekPos / 1000000;
        ui->slider_videoProcess->setValue(targetSec);
        // 更新时间显示...

        // 防抖定时器还在跑 → 继续等待
        if(m_seekTimer->isActive()) return;

        // seek已发出，检查实际播放位置是否追上目标
        qint64 currentSec = m_player->getCurrentTime() / 1000000;
        m_seekTicks++;
        // 播放位置到达目标(2秒容差) 或 超时(3秒=6周期) → 结束seeking
        if(qAbs(currentSec - targetSec) <= 2 || m_seekTicks > 6) {
            m_seeking = false;
        }
        return;
    }

    // 正常跟踪播放位置
    qint64 Sec = m_player->getCurrentTime() / 1000000;
    ui->slider_videoProcess->setValue(Sec);
}
```

**seek期间状态机**：

| 阶段 | `m_seeking` | 定时器行为 |
|------|-------------|-----------|
| 点击/释放轨道 | `true` | 手柄保持在目标位置 |
| seek防抖等待中 | `true` | 持续保持，不回拉 |
| seek已发出 | `true` | 检查播放位置是否追上目标 |
| 追上目标/超时 | `false` | 恢复正常跟踪 |

### 3.8 wait()超时保护

所有调用 `m_player->wait()` 的地方都改为 `wait(5000)`（5秒超时）：

```cpp
// playUrl - 切换视频
m_player->stop(false);
if(!m_player->wait(5000)) {
    qWarning() << "VideoPlayer thread did not finish within 5 seconds";
}

// on_pb_open_clicked - 打开本地文件
m_player->stop(false);
if(!m_player->wait(5000)) {
    qWarning() << "VideoPlayer thread did not finish within 5 seconds on open";
}

// ~PlayerDialog - 析构
m_player->stop(false);

    qWarning() << "VideoPlayer thread did not finish within 5 seconds in destructor";
```

有了 `interrupt_callback` 后实际不会超时，但作为安全网防止极端情况下主线程永久冻结。

## 四、播放器窗口生命周期管理

### 4.1 关闭窗口停止视频

**文件**：`playerdialog.h` / `playerdialog.cpp`

**问题**：PlayerDialog 没有 `closeEvent`，点 X 只是隐藏窗口，不调 `stop()`，音视频线程继续运行，声音不停。

**修复**：
```cpp
void PlayerDialog::closeEvent(QCloseEvent *event)
{
    m_userStopped = true;       // 阻止自动循环播放
    m_seekTimer->stop();
    m_seeking = false;

    if(m_player->isRunning()) {
        m_player->stop(false);
        if(!m_player->wait(5000)) {
            qWarning() << "VideoPlayer thread did not finish on close";
        }
    }
    slot_PlayerStateChanged(PlayerState::Stop);
    event->accept();  // 隐藏窗口（不销毁，CKernel仍持有指针可重新show）
}
```

### 4.2 再次点击GIF重新显示窗口

**文件**：`ckernel.cpp`

**问题**：关闭播放器窗口后，`m_playerDialog` 仍然存在（只是隐藏了）。再次点击GIF时，`slot_playRtmpUrl` 只调了 `playUrl()` 但没调 `show()`，窗口不出来。

## 七、踩坑记录

### 7.1 RTMP流seek的特殊性

RTMP是实时流媒体协议，`av_seek_frame` 对RTMP流的支持有限，可能返回失败。原始代码在seek失败时仍然设置 `seek_flag`，导致所有帧被跳过。**修复**：seek失败时不设flag，从当前位置继续播放。

### 7.2 FFmpeg阻塞I/O无法被quit中断

这是最核心的坑。FFmpeg的 `av_read_frame` 等函数是C语言级别的阻塞调用，完全不关心Qt的 `quit` 标志。必须通过 `interrupt_callback` 机制，让FFmpeg在阻塞时周期性调用回调函数检查是否应该中断。**修复**：注册 `decode_interrupt_cb`，quit时返回1。

### 7.3 音视频时钟不同步导致死等

FFmpeg播放器的音视频同步依赖 `video_clock` 和 `audio_clock` 两个时钟。seek时只重置video_clock不重置audio_clock，会导致视频线程死等音频时钟追赶。向前seek尤其严重（`video_pts > audio_pts` 永远成立）。**修复**：seek时同步重置 `audio_clock` 到目标位置。

### 7.4 QSlider + QTimer的交互问题

QSlider的 `sliderPressed`/`sliderReleased` 信号和自定义 `eventFilter` 点击轨道逻辑，与500ms定时器的 `setValue` 之间存在竞态。点击轨道时 `eventFilter` 消费了事件（`return true`），`sliderPressed` 不触发，`m_sliderPressed` 保持false，定时器就会拉回手柄。**修复**：引入 `m_seeking` 状态机，seek期间定时器保持手柄在目标位置。

### 7.5 QLabel scaledContents vs KeepAspectRatio

`QLabel::setScaledContents(true)` 会将图片拉伸填满整个Label区域，**不保持宽高比**，导致GIF变形。正确做法是使用 `QMovie::frameChanged` 信号，每帧手动 `scaled(size, Qt::KeepAspectRatio)` 后 `setPixmap`。

### 7.6 QGridLayout列宽不均

QGridLayout默认按各控件的 `sizeHint` 分配空间，当窗口宽度超出所有控件的minimumSize之和时，多余空间可能不均匀分配。**修复**：对每列设置相同的 `setColumnStretch(col, 1)`，并使用 `setFixedSize` 确保格子大小不变。
