# 视频热度推荐功能完整方案整理

## 一、整体业务流程概述

实现**按视频热度（点击量 hotdegree）给用户推送未浏览过的视频缩略图**，客户端提供自定义视频预览控件：鼠标移入播放预览动图、移出停止、点击触发视频播放信号；服务端负责数据库筛选、推送协议封装、记录已推送视频。

整体链路：客户端发起推荐请求 → 服务端数据库过滤未读视频（无新视频则清空用户记录）→ 按热度取 Top10 视频 → 入库记录已推送视频 → 封装协议结构体下发缩略图信息 → 客户端解析渲染预览控件。

## 二、数据库设计与 SQL 逻辑

### 1. 涉及数据表

- `t_VideoInfo`：视频总信息表，存储所有视频基础数据
    
    关键字段：`videoId`(视频 ID)、`picName`(缩略图文件名)、`picPath`(缩略图路径)、`rtmp`(视频流后缀路径)、`hotdegree`(热度 / 点击量)
- `t_UserRecv`：用户已接收视频记录表，记录该用户已经推送过的视频，避免重复推荐
    
    关键字段：`userId`(用户 ID)、`videoId`(已推送视频 ID)

### 2. 步骤 1：判断用户是否还有未推送的新视频

#### 功能逻辑

查询当前用户**从未接收过**的视频总数；若数量 = 0，说明全部视频都已推送，清空该用户的推送记录，重新一轮推荐。

#### SQL 语句（示例用户 ID=5）

sql

```
-- 查询用户未接收视频总数
select count(videoId) 
from t_VideoInfo 
where t_VideoInfo.videoId not in (
    select t_UserRecv.videoId 
    from t_UserRecv 
    where t_UserRecv.userId = 5
);

-- 无新视频时，清空该用户已推送记录
delete from t_UserRecv where userId = 5;
```

### 3. 步骤 2：按热度拉取用户未读 Top10 视频

#### 功能逻辑

过滤用户已接收视频，按热度`hotdegree`降序排序，分页取前 10 条，获取视频缩略图、播放流相关信息。

#### SQL 语句（示例用户 ID=5）

sql

```
select videoId, picName, picPath, rtmp, hotdegree 
from t_VideoInfo 
where t_VideoInfo.videoId not in (
    select t_UserRecv.videoId 
    from t_UserRecv 
    where t_UserRecv.userId = 5
) 
order by hotdegree desc 
limit 0,10; -- limit 偏移量,条数，实现分页，0起始取10条
```

### 4. 步骤 3：记录本次推送视频到用户接收表

将本次推荐给用户的 10 条视频 ID、用户 ID 插入`t_UserRecv`，标记为已推送，下次不再重复推荐。

sql

```
insert into t_UserRecv values(%d ,%d); -- 入参：userId, videoId
```

## 三、网络通信协议结构体（C/C++ 客户端 <-> 服务端）

### 1. 协议包类型定义

- `DEF_PACK_DOWNLOAD_RQ`：客户端**推荐视频请求包**
- `DEF_PACK_DOWNLOAD_RS`：服务端**推荐视频回复包**（携带缩略图、播放地址信息）

### 2. 请求结构体 STRU_DOWNLOAD_RQ（客户端发给服务端）

客户端请求获取推荐视频列表时发送

c

运行

```
typedef struct STRU_DOWNLOAD_RQ
{
    // 构造函数初始化包类型、用户ID
    STRU_DOWNLOAD_RQ()
    {
        m_nType = DEF_PACK_DOWNLOAD_RQ;
        m_nUserId = 0;
    }
    PackType m_nType;    // 包类型标识
    int      m_nUserId;  // 当前请求的用户ID
} STRU_DOWNLOAD_RQ;
```

### 3. 回复结构体 STRU_DOWNLOAD_RS（服务端下发给客户端）

每条视频信息封装一个该结构体，下发给客户端渲染预览控件

c

运行

```
typedef struct STRU_DOWNLOAD_RS
{
    // 构造函数初始化，清空字符串缓冲区
    STRU_DOWNLOAD_RS()
    {
        m_nType = DEF_PACK_DOWNLOAD_RS;
        m_nFileId = 0;
        memset(m_szFileName, 0, MAX_PATH);
        memset(m_rtmp, 0, MAX_PATH);
    }
    PackType m_nType;            // 包类型标识
    int      m_nFileId;          // 文件ID
    int64_t  m_nFileSize;        // 文件大小
    int      m_nVideoId;         // 视频ID（对应数据库videoId）
    char     m_szFileName[MAX_PATH]; // 缩略图文件名(picName)
    char     m_rtmp[MAX_PATH];   // 视频流后缀路径（如/1/103.MP3）
} STRU_DOWNLOAD_RS;
```

#### RTMP 播放地址拼接规则

客户端本地拼接完整播放地址：

`rtmp://服务器IP/vod/` + `m_rtmp`

> 项目固定 app 名称：vod

### 4. 数据下发流程

1. 服务端查询出 Top10 视频，循环生成 10 个`STRU_DOWNLOAD_RS`结构体；
2. 依次封装网络包下发给客户端；
3. 文件传输逻辑和普通文件上传 / 下载流程一致（分包传输、文件头 + 文件块）。

## 五、完整业务执行总流程

1. **客户端请求**：构造`STRU_DOWNLOAD_RQ`，带上用户 ID 发送给服务端；
2. **服务端数据库预处理**
    
    - 执行 count 查询，统计用户未推送视频数量；
    - 数量 = 0 → `delete`清空该用户`t_UserRecv`记录；
    
3. **拉取推荐列表**：执行热度排序 limit 10 的 SQL，获取视频缩略图、rtmp、videoId；
4. **标记已推送**：循环插入`t_UserRecv`，记录本次推送的 userID+videoId；
5. **下发视频信息**：每条视频封装`STRU_DOWNLOAD_RS`，通过网络分包下发；
6. **客户端渲染控件**
    
    - 解析每个回复包，读取缩略图路径加载 QMovie；
    - 创建`QMyMovieLabel`控件，调用`setMovie()`、`setRtmpUrl()`绑定数据；
    - 布局批量展示多个预览控件；
    
7. **用户交互**
    
    - 鼠标移入单个控件：自动播放缩略动图预览；
    - 鼠标移出：停止动图；
    - 点击控件：触发`SIG_labelClicked`信号，上层读取存储的 rtmp 地址，播放完整视频。

# 视频热度推荐完整系统全流程整理（客户端 Qt + 服务端 TCP 后台）

## 整体架构拆分

1. **客户端（Qt）**：OnlineDialog 网络接收分发 + QMyMovieLabel 自定义视频预览控件
2. **服务端（C++ TCP）**：TcpKernel 处理客户端请求、MySQL 数据库筛选视频、封装协议下发缩略图文件
3. **数据库**：`t_VideoInfo`（全量视频）、`t_UserRecv`（用户已推送记录）
4. **通信协议**：`STRU_DOWNLOAD_RQ`（客户端请求）、`STRU_DOWNLOAD_RS`（服务端视频信息头）、`STRU_FILEBLOCK_RQ`（文件分片包）

## 1. 网络入口：OnlineDialog::slot_ReadyData 网络数据分发

### 功能

TCP 收到服务端二进制数据，读取包类型`nType`，分发到对应业务处理函数

cpp

运行

```
void OnlineDialog::slot_ReadyData(char *buf, int nlen)
{
    int nType = *(int*) buf;
    switch (nType)
    {
        case DEF_PACK_LOGIN_RS:
            slot_loginRs(buf, nlen);
            break;
        case DEF_PACK_REGISTER_RS:
            slot_registerRs(buf, nlen);
            break;
        case DEF_PACK_UPLOAD_RS:
            slot_uploadRs(buf, nlen);
            break;
        // 热度推荐视频缩略图回复包
        case DEF_PACK_DOWNLOAD_RS:
            slot_downloadRs(buf, nlen);
            break;
    }
    delete [] buf; // 释放接收缓冲区
}
```

## 2. 推荐视频处理函数：slot_downloadRs（核心客户端逻辑）

### 作用

解析服务端下发的`STRU_DOWNLOAD_RS`视频信息头，创建本地缓存目录、构建`FileInfo`视频数据对象、拼接完整 RTMP 播放地址、打开本地文件准备接收缩略图分片、存入全局 map 管理视频数据。

cpp

运行

```
void OnlineDialog::slot_downloadRs(char *buf, int nlen)
{
    // 1. 强制转换为视频信息回复协议结构体
    STRU_DOWNLOAD_RS *rs = (STRU_DOWNLOAD_RS *)buf;

    // 2. 封装本地业务数据对象 FileInfo
    FileInfo *info = new FileInfo;
    info->videoId = rs->m_nVideoId;
    info->fileId = rs->m_nFileId;
    info->fileName = rs->m_szFileName;

    // 3. 创建本地temp缓存目录，存放缩略图gif
    QDir dir;
    if (!dir.exists(QDir::currentPath() + "/temp/"))
    {
        dir.mkpath(QDir::currentPath() + "/temp/");
    }

    // 4. 拼接缩略图本地完整路径
    info->filePath = QString("./temp/%1").arg(rs->m_szFileName);
    info->filePos = 0;          // 文件当前读写偏移
    info->fileSize = rs->m_nFileSize; // 缩略图总大小

    // 5. 拼接完整RTMP播放地址
    // DEF_SERVER_IP 为服务端固定IP，项目固定app名vod
    info->rtmpUrl = QString("rtmp://%1/vod/%2")
        .arg(DEF_SERVER_IP)
        .arg(rs->m_rtmp);

    // 6. 打开本地文件，准备写入分片缩略图
    info->pFile = new QFile(info->filePath);
    if (info->pFile->open(QIODevice::WriteOnly))
    {
        // 存入全局map：videoId -> FileInfo，用于控件绑定
        m_mapVideoIDTOFileInfo[info->videoId] = info;
    }
    else
    {
        // 文件打开失败，释放内存
        delete info;
    }
}
```

# 第二部分：服务端 TCP 后台逻辑（TcpKernel.cpp）

## 1. 客户端推荐请求入口：DownloadRQ

客户端发送`STRU_DOWNLOAD_RQ`请求推荐视频，服务端入口函数

cpp

运行

```
void TcpKernel::DownloadRQ(int clientfd ,char* szbuf,int nlen)
{
    printf("clientfd:%d DownloadRq\n", clientfd);
    // 解析客户端请求包
    STRU_DOWNLOAD_RQ * rq = (STRU_DOWNLOAD_RQ *)szbuf;
    list<FileInfo*> fileList;
    // 核心：数据库查询当前用户可推荐视频列表
    GetFileList(fileList ,rq->m_nUserId);

    // 循环下发每个视频：先下发文件头STRU_DOWNLOAD_RS，再分片下发缩略图二进制
    while( fileList.size() > 0 )
    {
        FileInfo* info = fileList.front();
        fileList.pop_front();

        // 1. 封装视频信息回复包 STRU_DOWNLOAD_RS
        STRU_DOWNLOAD_RS rs;
        strcpy( rs.m_rtmp , info->m_szRtmp );
        rs.m_nFileId = info->m_nFileID;
        rs.m_nVideoId = info->m_VideoID;
        rs.m_nFileSize = info->m_nFileSize;
        strcpy( rs.m_szFileName , info->m_szFileName );
        // 发送视频信息头
        m_tcp->SendData( clientfd , (char*)&rs , sizeof(rs) );

        // 2. 打开缩略图本地文件，分片下发文件内容
        info->pFile = fopen( info->m_szFilePath , "r" );
        if( info->pFile )
        {
            while(1)
            {
                STRU_FILEBLOCK_RQ blockrq;
                // 读取分片数据
                int64_t res = fread( blockrq.m_szFileContent, MAX_CONTENT_LEN , 1 , info->pFile );
                blockrq.m_nBlockLen = res;
                info->m_nPos += res;
                // 发送分片包
                m_tcp->SendData( clientfd , (char*)&blockrq , sizeof(blockrq) );

                // 文件读取完成，关闭释放
                if( info->m_nPos >= info->m_nFileSize )
                {
                    fclose(info->pFile);
                    delete info;
                    info = NULL;
                    break;
                }
            }
        }
    }
}
```

## 2. 数据库核心函数 GetFileList（热度筛选视频逻辑）

### 4 大执行步骤

1. 查询用户未接收视频总数，无新视频则清空用户已推送记录；
2. 按热度`hotdegree`降序，取 Top10 未推送视频；
3. 遍历 SQL 结果，封装服务端`FileInfo`对象存入列表；
4. 将本次推荐视频写入`t_UserRecv`，标记为已推送。

cpp

运行

```
void TcpKernel::GetFileList( list<FileInfo*>& fileList , int userid )
{
    char sqlStr[1024] = "";
    int nCount = 0;
    list<string> resList;

    // ========== 步骤1：查询用户未接收视频数量 ==========
    sprintf(sqlStr,
        "select count(videoId) from t_VideoInfo "
        "where t_VideoInfo.videoId not in ("
        "select t_UserRecv.videoId from t_UserRecv where t_UserRecv.userId = %d);",
        userid);
    if( !m_sql->SelectMysql(sqlStr , 1 , resList) )
    {
        printf("SelectMysql error : %s\n", sqlStr);
        return;
    }
    if( resList.size() == 0 ) return;
    nCount = atoi( resList.front().c_str() );

    // 无未推送视频，清空该用户全部推送记录，重置推荐池
    if( nCount == 0 )
    {
        sprintf(sqlStr, "delete from t_UserRecv where userId = %d;", userid);
        if( !m_sql->UpdataMysql(sqlStr) )
        {
            printf("UpdataMysql error : %s\n", sqlStr);
            return;
        }
    }

    // ========== 步骤2：按热度hotdegree降序，取Top10未推送视频 ==========
    resList.clear();
    sprintf(sqlStr,
        "select videoId ,picName ,picPath ,rtmp ,hotdegree "
        "from t_VideoInfo "
        "where t_VideoInfo.videoId not in ("
        "select t_UserRecv.videoId from t_UserRecv where t_UserRecv.userId = %d) "
        "order by hotdegree desc limit 0,10;",
        userid);
    if( !m_sql->SelectMysql(sqlStr , 5 , resList) )
    {
        printf("SelectMysql error : %s\n", sqlStr);
        return;
    }

    // ========== 步骤3：遍历SQL结果，封装FileInfo对象 ==========
    int nFileId = 1;
    for ( int i = 0 ; i < (int)(resList.size()/4) ; ++i )
    {
        FileInfo * info = new FileInfo;
        info->m_nPos = 0;
```

# 第五部分：完整端到端业务执行流程

1. **客户端发起请求**
    
    界面触发推荐逻辑，构造`STRU_DOWNLOAD_RQ`，携带当前登录 userId 发送 TCP 给服务端。
2. **服务端接收请求 DownloadRQ**
    
    调用`GetFileList`执行数据库筛选逻辑：
    
    - 判断用户是否还有未推送视频；无则清空`t_UserRecv`记录；
    - 按热度降序取 Top10 视频；
    - 读取缩略图文件大小，封装`FileInfo`；
    - 插入`t_UserRecv`标记已推送。
    
3. **服务端下发视频数据**
    
    循环遍历视频列表，对每条视频：
    
    1. 发送`STRU_DOWNLOAD_RS`视频信息头；
    2. 读取本地缩略图 GIF，拆分`STRU_FILEBLOCK_RQ`分片包下发二进制；
    
4. **客户端网络分发 slot_ReadyData**
    
    收到`DEF_PACK_DOWNLOAD_RS`包，调用`slot_downloadRs`处理：
    
    - 创建本地`./temp/`缓存目录；
    - 构建`FileInfo`对象，拼接完整`rtmp://ip/vod/xxx`播放地址；
    - 打开本地文件，等待接收分片写入 GIF。
    
5. **客户端渲染预览控件**
    
    缩略图文件下载完成后，创建`QMyMovieLabel`控件：
    
    - `setMovie(QMovie)`加载本地 GIF；
    - `setRtmpUrl(info->rtmpUrl)`绑定播放地址；
    - 批量布局展示多个预览控件。
    
6. **用户交互逻辑**
    
    - 鼠标移入控件：`enterEvent`启动 GIF 预览；
    - 鼠标移出控件：`leaveEvent`停止 GIF；
    - 点击控件：触发`SIG_labelClicked`信号，上层读取 rtmpUrl 播放完整 RTMP 视频。

## （一）GetFileList 函数数据库查询字段不匹配问题

### 原错误点

1. 最初 SQL 查询 5 个字段 `videoId ,picName ,picPath ,rtmp ,hotdegree`，`SelectMysql` 参数传 `5`；
2. 截图中修改后 SQL 只查询 4 个字段 `videoId ,picName ,picPath ,rtmp`，但 `SelectMysql` 第二个参数仍写死 `4` 这里是匹配的，但**循环取值逻辑有隐患**：
    
    - `int nSize = resList.size()/4;` 依赖字段数量，后续改 SQL 字段数会直接崩溃；
    - 原代码读取 hotdegree 后丢弃，新版 SQL 直接去掉 hotdegree，逻辑无报错，但要注释说明。
    
3. 内存泄漏风险：SQL 异常 return 时，已 new 的 FileInfo 未释放。

```
void TcpKernel::GetFileList( list<FileInfo*>& fileList , int userid )
{
    char sqlStr[1024] = "";
    int nCount = 0;
    list<string> resList;

    // ========== 步骤1：查询用户未接收视频总数 ==========
    sprintf(sqlStr,
        "select count(videoId) from t_VideoInfo "
        "where t_VideoInfo.videoId not in ("
        "select t_UserRecv.videoId from t_UserRecv where t_UserRecv.userId = %d);",
        userid);
    if( !m_sql->SelectMysql(sqlStr , 1 , resList) )
    {
        printf("SelectMysql error : %s\n", sqlStr);
        return;
    }
    if( resList.size() == 0 ) return;
    nCount = atoi( resList.front().c_str() );

    // 无未推送视频，清空该用户全部推送记录，重置推荐池

        sprintf(sqlStr, "delete from t_UserRecv where userId = %d;", userid);
        if( !m_sql->UpdataMysql(sqlStr) )
        {
            printf("UpdataMysql error : %s\n", sqlStr);
            return;
        }

        return;

    int nFileId = 1;

            continue;
        }
        fseek( info->pFile , 0 , SEEK_END );
        info->m_nFileSize = ftell(info->pFile);
        fseek( info->pFile , 0 , SEEK_SET );

        fileList.push_back(info);
```

```
void TcpKernel::DownloadRQ(int clientfd ,char* szbuf,int nlen)
{
    printf("clientfd:%d DownloadRq\n", clientfd);
    // 解析客户端请求包
    STRU_DOWNLOAD_RQ * rq = (STRU_DOWNLOAD_RQ *)szbuf;
    list<FileInfo*> fileList;
    // 数据库查询可推荐视频
    GetFileList(fileList ,rq->m_nUserId);

    // 循环下发每条视频缩略图
    while( fileList.size() > 0 )
    {
        FileInfo* info = fileList.front();
        fileList.pop_front();

        // 1. 封装视频信息头包 STRU_DOWNLOAD_RS
        STRU_DOWNLOAD_RS rs;
        strcpy( rs.m_rtmp , info->m_szRtmp );
        rs.m_nFileId = info->m_nFileID;
        rs.m_nVideoId = info->m_VideoID;
        rs.m_nFileSize = info->m_nFileSize;
        strcpy( rs.m_szFileName , info->m_szFileName );
        m_tcp->SendData( clientfd , (char*)&rs , sizeof(rs) );

        // 2. 打开缩略图文件
        info->pFile = fopen( info->m_szFilePath , "r" );
        if( info->pFile == nullptr )
        {
            printf("缩略图文件打开失败:%s\n", info->m_szFilePath);
            delete info; // 文件打不开直接释放
            continue;
        }

        // 3. 循环分片发送文件二进制
        bool bFinish = false;
        while(!bFinish)
        {
            STRU_FILEBLOCK_RQ blockrq;
            // 修复fread参数顺序错误
            int64_t res = fread(blockrq.m_szFileContent, 1, MAX_CONTENT_LEN, info->pFile);
            blockrq.m_nBlockLen = res;
            info->m_nPos += res;
            blockrq.m_nFileId = info->m_VideoID;
            blockrq.m_nUserId = rq->m_nUserId;

            m_tcp->SendData( clientfd , (char*)&blockrq , sizeof(blockrq) );

            // 文件全部发送完成
            if( info->m_nPos >= info->m_nFileSize )
            {
                fclose(info->pFile);
                delete info;
                info = nullptr;
                bFinish = true;
            }
        }
    }

    // 兜底：如果fileList还有残留节点，全部释放防止内存泄漏
    for(auto p : fileList)
    {
        delete p;
    }
    fileList.clear();
}
```

## 二、核心逻辑拆解

1. **时间单位换算**

4. **播放结束自动停止容错**

## 四、配套联动逻辑说明

1. **定时器 m_Timer**：一般设置 100ms~1000ms 定时触发，持续同步播放进度；
2. **slot_videoSliderValueChanged**：拖动滑块时，将 value（秒）转为微秒调用播放器 seek 接口，实现拖拽跳转；
3. **slot_PlayerStateChanged**：统一处理播放 / 暂停 / 停止的 UI 切换（播放按钮文字、进度条可用性等）；
4. 与之前 RTMP 播放模块联动：点播 / 直播加载完成后，初始化进度条最大值，启动定时器刷新 UI。

## 五、完整优化后定时器代码

cpp

运行

```
if (QObject::sender() == m_Timer)
{
    // 仅播放状态刷新进度
    if (m_Player->m_playerState != PlayerState::Playing)
        return;

    qint64 totalUs = m_Player->getCurrentTime();
    qint64 totalSec = totalUs / 1000000;

    // 更新进度滑块
    ui->slider_videoProcess->setValue(totalSec);

    // 格式化时分秒，原生arg补零
    qint64 h = totalSec / 3600;
    qint64 m = (totalSec / 60) % 60;
    qint64 s = totalSec % 60;
    QString timeStr = QString("%1:%2:%3")
        .arg(h, 2, 10, QChar('0'))
        .arg(m, 2, 10, QChar('0'))
        .arg(s, 2, 10, QChar('0'));
    ui->lb_videoCurrentTime->setText(timeStr);

    // 视频播完自动停止，兼容取整误差
    int curVal = ui->slider_videoProcess->value();
    int maxVal = ui->slider_videoProcess->maximum();
    if ((curVal == maxVal || curVal + 1 == maxVal) && m_Player->m_playerState == PlayerState::Stop)
    {
        slot_PlayerStateChanged(PlayerState::Stop);
    }
}
```
