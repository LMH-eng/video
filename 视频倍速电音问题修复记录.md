# 视频倍速播放电音问题修复记录

## 问题描述
加速播放时（1.5x/2.0x）音频出现"电音"（哇啦声/机械音），0.5x倍速正常。

## 根因分析
1. **格式不匹配**：atempo 滤镜输出 FLT（format=3），但 swr_ctx 配置为 FLTP→S16（format=8→1），格式不匹配导致音频数据被错误解析
2. **rubberband 相位问题**：rubberband 滤镜在部分人声素材上容易出现"相位叠加电音感"
3. **多余帧丢弃**：atempo 多帧输出时被丢弃，导致音频片段被截断

## 修复方案

### 1. 仅使用 atempo 滤镜（去掉 rubberband 路径）
文件：`videoplayer.cpp` 第151-180行
```cpp
// 只使用 atempo，避免 rubberband 在部分素材上出现相位叠加/电音感
// atempo 滤镜支持范围是 0.5 ~ 2.0，超过2.0需要串联多个
while (remaining > 2.0) {
    // 创建 atempo=2.0 滤镜
    remaining /= 2.0;
}
// 添加最后一个 atempo
if (fabs(remaining - 1.0) > 0.001) {
    // 创建 atempo=remaining 滤镜
}
```

### 2. 统一 SDL 输出格式为 AUDIO_S16SYS
文件：`videoplayer.cpp` 第441行
```cpp
wanted_spec.format = AUDIO_S16SYS;  // 统一使用 S16 输出
```

### 3. 固定 out_frame.format 为 AV_SAMPLE_FMT_S16
文件：`videoplayer.cpp` 第457行
```cpp
m_videoState.out_frame.format = AV_SAMPLE_FMT_S16;
```

### 4. 在滤镜图中添加 aformat 强制输出 S16
文件：`videoplayer.cpp` 第182-200行
```cpp
// 添加 aformat 滤镜，强制输出 S16 格式

if (aformat_filter) {

             is->pAudioCodecCtx->sample_rate, ...);
    // 创建 aformat 滤镜并链接
}
```

### 5. 删除多余帧丢弃逻辑
文件：`videoplayer.cpp` 第1059行
```cpp
// 成功输出，直接使用第一帧（不丢弃多余帧）
filtered_frame = temp_frame;
```

## 修复后的音频管道
```
解码器(FLTP) → atempo(FLT) → aformat(S16) → memcpy → SDL(S16)
```

## 关键改进
1. **彻底绕过 swr_ctx**：aformat 在滤镜图内部完成 FLT→S16 转换，避免格式不匹配
2. **统一格式**：SDL 输出和滤镜输出均为 S16，避免格式协商带来的隐式失真
3. **保留所有帧**：不丢弃 atempo 输出的多余帧，避免音频片段被截断

## 反思
这次修复的关键在于：
1. **格式追踪**：通过日志 `in_fmt=3` 发现 atempo 实际输出格式与 swr_ctx 配置不匹配
2. **滤镜链优化**：利用 FFmpeg 滤镜图内部的 aformat 完成格式转换，比手动 swr_ctx 更可靠
3. **简化管道**：减少转换环节，降低失真风险

之前的尝试走了弯路，主要因为没有正确追踪实际的数据格式转换链路，以及对 FFmpeg 滤镜机制理解不够深入。
