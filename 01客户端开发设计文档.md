## 一、开发理念与质量标准（工业级）

### 1.1 用户核心理念（必须贯穿始终）

| 原则 | 说明 |
|------|------|
| **步步确认** | 每个功能模块实现前必须有计划，实现后必须验证，不允许跳跃式开发 |
| **企业级对标** | 输入校验、错误提示、异常处理、用户体验必须达到商业软件标准 |
| **可测试性** | 每个模块独立可测，不依赖其他未完成模块 |
| **文档先行** | 设计先于代码，笔记记录完整决策过程 |
| **零容忍Bug** | 编译警告尽量消除，运行时崩溃必须根因分析 |

### 1.2 代码规范

- 变量命名：`m_`前缀成员变量，`ui->`访问UI控件
- 信号槽：`SIG_`前缀信号，`slot_`前缀槽函数
- 资源释放：析构函数中逐一释放，指针置nullptr
- 线程安全：UI操作必须在主线程，网络IO在子线程（moveToThread）
- 协议结构体：构造函数初始化所有字段，memset清零

## 五、关键技术决策记录

### 5.1 为什么用 moveToThread 而不是 QRunnable/QConcurrent？

| 方案 | 优点 | 缺点 | 选择 |
|------|------|------|------|
| moveToThread | 对象生命周期可控，支持信号槽跨线程 | 需要手动管理线程 | ✅ 采用 |
| QConcurrent::run | 简单一行调用 | 无法取消，无法汇报进度 | ❌ |
| QRunnable+QThreadPool | 可复用线程池 | 信号槽不支持跨线程 | ❌ |

**结论**: UploadWorker 继承 QObject，moveToThread 到专用 QThread，通过信号槽汇报进度。

### 5.2 为什么上传用TCP而不是HTTP？

- 课程要求使用自定义二进制协议（packdef.h struct）
- TCP 长连接避免每次上传重建连接开销
- 分片上传需要精确控制每个包的边界
- 后续可扩展：断点续传、加密传输

### 5.3 文件块大小为什么是 4KB (_DEF_BUFFER)？

- 匹配操作系统内存页大小
- TCP MTU 约1500字节，4KB = 约3个TCP包，效率适中
- 太小：包数量多，协议头开销大
- 太大：单次阻塞时间长，进度更新不及时

### 5.4 GIF 生成为什么用两步法？

- 第一步：FFmpeg C API 解码抽I帧 → 保存为 jpg（精确控制帧选择）
- 第二步：QProcess 调用 ffmpeg CLI 合成 GIF（ffmpeg CLI 的 GIF 编码器比 libavformat 更成熟）
- 如果纯用 FFmpeg C API 生成 GIF，需要引入额外的 GIF 库或处理 palette 问题

## 六、FFmpeg 版本兼容性预案

### 当前代码使用的旧API（FFmpeg 3.x）：

```cpp
av_register_all();                          // FFmpeg 4.0+ 废弃
avcodec_decode_video2();                    // FFmpeg 3.1+ 废弃
avpicture_get_size() / avpicture_fill();    // FFmpeg 4.0+ 废弃
avcodec_close();                            // FFmpeg 4.0+ 废弃
```

### 如果检测到 FFmpeg 4.x+：

```cpp
#if LIBAVCODEC_VERSION_INT < AV_VERSION_INT(58, 10, 100)
    av_register_all();  // 仅旧版需要
#endif

// 新版解码循环
while(av_read_frame(pFormatCtx, packet) >= 0) {
    ret = avcodec_send_packet(pCodecCtx, packet);  // 新API
    while(ret >= 0) {
        ret = avcodec_receive_frame(pCodecCtx, pFrame);  // 新API
        if(ret == AVERROR(EAGAIN) || ret == AVERROR_EOF) break;
        // 处理 frame...
    }
}
```

**注意**: 当前 uploaddialog.cpp 已经使用了新的 send_packet/receive_frame API（第239-249行），但仍然调用了 `av_register_all()`（第166行）。需要在编译时根据版本条件处理。
